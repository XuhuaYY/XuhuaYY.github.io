大O表示法

为什么需要大O表示法

  • 算法A比算法B快两倍，这种说法没有意义，因为没有控制数据项的个数

  • 采用 算法的效率：算法时间与数据量的大小之间的变化进行评估是合理的，这就是大O表示法

大O表示法的种类
  1. O(1)表示算法时间随着数据量的增大是一个常数集合
  2. O(log(n))表示算法时间随着数据量呈现出log函数的关系
  3. O(n)表示算法时间随着数据量是线性的
  4. O(nlog(n))表示算法时间随着数据量是线性和对数乘积
  5. O()表示算法时间随着数据量是平方关系
  6. O()表示算法时间随着数据量是指数关系的
大O表示法的推导

①用常量代替所有的加法常量

②修改的多项式函数中，只保留最高级阶项

③系数不唯一1，化为1

function ArrayList() {
    //属性
    this.array = []
    
    //方法
    //[方法1：插入]
    ArrayList.prototype.insert = function(item) {
        this.array.push(item);
    }
    
    //[方法2：返回字符串]
    ArrayList.prototype.toString = function() {
        return this.array.join('-');
    }
    
    
    //[排序的方法如下]
    //a.冒泡排序
    /*
    	冒泡排序的思路是：列表中两两相邻的数据进行比较，大的向后排[升序]，反之小的向后排[降序]，一次遍历可以得到最大的值放在后面；再通过以上的方式找到次最大的，依次下去。
    	
    */
    ArrayList.prototype.swap(m, n) {
        let midVal = this.array[m];
        this.array[m] = this.array[n];
        this.array[n] = midVal;
    }
    
    ArrayList.prototype.bubbleSorting = function() {
    	for(let j=this.array.length-1;j>1; j--) {
        	for(let i=0; i<j; i++) {
                if (this.array[i] > this.array[j]) {
                	swap(i, i+1);
                }
        	}
        }
    }
    
    
    //b.选择排序
    /*
    	选择排序的思路是：minIndex=0，列表中循环遍历与minIndex的数据比较，若小，则重新记录minIndex，遍历完整个列表，交换第minIndex个与第0个的数值，相当于找到最小的值；
    	之后minIndex=1，从第二个位置继续遍历查找，替换minIndex的位置；结束后交换minIndex与第1个的数值，相当于找到次小的值；
    	依次下去直到最后。
    */
    ArrayList.prototype.selectionSorting = function() {
 
        var length = this.array.length;
        for (let j=0; j<length-1; j++) {
            var minIndex =  j;
            for(let i=j; i<length; i++) {
                if (this.array[minIndex] > this.array[i]) {
                    minIndex = i; 
                } 
            }
            this.swap(j ,minIndex)
        }   
    }

    
    
    
    //插入排序
    //希尔排序
    //快速排序
    
    
    
}
