数组

（优点）

通过索引的方式直接访问每个元素，直观且快速。

（缺点）

  • 连续固定内存存储的局限性。连续内存空间，当前数组不满足容量，会自动扩容（自动申请更大的内存，2倍内存，再将原数组中的元素copy过去）
  • 操作成本高。在数组开头或中间插入元素，需要进行大量的元素位移。
链表

（特点）

  • 链表由节点组成，每个节点包括一个元素和一个指向下一个节点的引用组成。
  • 不在栈中存储，存储在堆中，所以是不连续的内存空间。

（优点）

  • （空间上）动态内存管理
  • （规模上）创建时不需要确定大小
  • 操作成本低。操作插入和删除，时间复杂度达到O(1)，相对数组效率高得多。

（缺点）

  • 节点不能通过索引的方式快速访问，需要从头开始访问，直到对应的节点。
  • 单链表可以轻松地访问下一个节点，但回到前一个节点是困难的。
单向链表的直观理解

将数组看成是流水线上的商品，看起来是连续的，固定的。

单向链表则是火车，由一个火车头（head），连接着数个车厢（node）组成。而每节车厢对应的有商品（item）和连接下一节车厢的挂钩（next）组成。

链表的数据结构

head属性指向链表的第一个节点。

每个节点对象中有两个属性，一个是元素（item），另一个是下一个节点的引用（next）。

最后一个节点指向无意义的地址，也就是next设置为null

链表中的常见操作

  • append(element) 向链表尾部添加一个新的项。

  • toString() 以String的方式依次返回链表中的元素。 重写Node原型中的toString()方法。

  • insert(position, element) 向链表的特定位置插入一个新的项。

  • getDate(position) 获取对应位置的元素。

  • update(position, element) 更新某个位置元素值。

  • indexOf(element) 返回该元素对应在链表中的索引，若没有该元素返回-1。

  • removeAt(position) 从链表某个位置删除一个节点。

  • remove(element) 从链表中查找某个元素，并删除该节点。

  • isEmpty() 如果链表是否为空，空返回true，否则反之。

  • size() 返回链表的节点（元素）个数。

⭐append方法

需要考虑几个方面：

该链表是否为空？

情况一：空，则this.head直接引用新的节点

情况二：非空，则按照链表的长度this.length从头开始遍历到最后一个节点，在尾节点引用新的节点。

append方法实现代码如下所示：

LinkedList.prototype.append = function(element) {
   //创建新节点
   newNode = new Node(element);
    /*
        考虑两种情况
        空链表/非空链表
    */
   //空链表 将新节点添加到head的引用
   if (this.length == 0) {
    this.head = newNode;
   }
   //非空链表 将新节点添加到最后一个节点；通过该节点的next引用新的节点
   else {
    let currentNode = this.head;
    while(currentNode.next != null) {
        currentNode = currentNode.next;
    }
    currentNode.next = newNode;
   }

   this.length++
    /*
        释放操作引用值
    */
   currentNode = null;
   return newNode;
}

⭐⭐insert方法

当前位置插入元素有两种实现方法（方法一是使用两个操作节点，方法二是使用一个操作节点）

需要思考几个方面：

1.位置越界[position小于0或是大于(index+1)，即length]

Ⅰ.【方法一】：考虑两种情况：

a.position=0,在第一个节点的位置插入元素

由于第一个节点，被this.head引用，所以直接通过操作新节点与this.head即可插入元素，步骤如下：

①newNode.next = this.head; //将新节点的next引用到第一个节点（this.head此时为右值——引用值，也就是第一个节点的堆内存地址）

②this.head=newNode;//将链表的head引用到新的节点，this.head代表的是在栈中的引用量，本来存储的是第一个节点的地址值，现在存储的是新节点的地址值。

b.position=n,则在第n个节点插入元素（n>0）

①newNode.next = currentNode;

②lastNode.next=newNode;

需要记录当前操作节点和上一个操作节点，再将新节点引用当前节点，上一个节点引用新节点实现插入到当前位置的操作。

insert方法（方法一）实现代码如下所示：

LinkedList.prototype.insert = function(element, position) {
    newNode = new Node(element);
    /*
    1.考虑越界,设置边界
    position不能小于0
    position不能大于index+1，对应的是length

    2.为首位插入
    操作的节点为this.head
    因此实现的步骤：先将新节点引用到第一个节点；再将头结点引用到新的节点

    3.非首位插入
    操作的节点是currentNode
    因此实现的步骤是：先将新节点引用到当前节点；再将前一个节点引用到当前节点
    */
   //1
   if (position < 0 || position > this.length) {
        return false;
   }
   else {
        let currentNode = this.head;
        let lastNode = null;
        let index = 0;
        //2
        if (position == 0) {
            newNode.next = this.head;
            this.head = newNode;
        }
        //3
        else{
            while(index++ < position) {
                lastNode = currentNode;
                currentNode = currentNode.next;
            }

            newNode.next = currentNode;
            lastNode.next = newNode;
        }
        this.length++;
        /*
            释放操作引用量
        */
        currentNode = null;
        lastNode = null;
        return true;

   }
}

Ⅱ.【方法二】

insert方法（方法二）实现代码如下所示：

LinkedList.prototype.customInsert = function(element, position) {
       newNode = new Node(element);
       /*
           1.考虑越界:(position<0 或是position>index+1 or length)

           2.为首位插入
           操作的节点为this.head
           因此实现的步骤：先将新节点引用到第一个节点；再将头结点引用到新的节点

           3.非首位插入
           操作的节点是lastNode
           因此实现的步骤是：先将新节点引用到当前节点(通过上一个节点获得)；再将上一个节点引用新节点
       */
      //1
       if (position < 0 || position > this.length) {
           return false;
       }
       else {
           //2
           if (position == 0) {
               newNode.next = this.head;
               this.head = newNode;
           }
           //3
           else {
               let lastNode = this.head;
               let index = 0;
               while(++index < position) {
                   lastNode = lastNode.next;
               }
               newNode.next = lastNode.next;
               lastNode.next = newNode;
           }
       }
       this.length++;
       /*
           释放操作引用值
       */
       lastNode = null;
       return true;
   }

insert方法的小结：

1、在首节点位置插入与非首节点插入的操作不同，需要分情况考虑。

2、涉及到位置的需要考虑越界，设置安全边界。

3、使用当前和上一个操作节点引用值来插入新节点更好理解；但也可以只通过上一个节点的引用值操作。其中，节点能很好地找到下一个节点，但不容易获取到上一个节点，这也是单链表的特点。

⭐getData方法

LinkedList.prototype.getData = function(position) {
    /*
        a.考虑越界，设置安全边界
            1.position为负数，或是超过最大索引，即length-1
            2.链表为空,this.length == 0

        b.从头节点开始按照position依次获取到当前节点，再得到对应的数值返回
        */
   if (position < 0 || position > this.length - 1 || this.length == 0) {
        return undefined;
   }
   else {
        let currentNode = this.head;
        let index = 0;
        while(index++ < position) {
            currentNode = currentNode.next;
        }
        return currentNode.element;
   }
}

⭐removeAt方法

删除某位置上的节点，相当于是前一个节点引用到下一个节点上，跳过当前节点。（不需要考虑该删除节点内存的释放）

考虑情况如下：

1、考虑越界

2、头结点删除 this.head = null;

3、非头结点删除 this.lastNode.next = this.currentNode.next;

LinkedList.prototype.removeAt = function(position) {
    /*
        a.考虑越界，设置安全边界
        b.头结点删除
        c.非头结点删除
    */
   //1
    if (position < 0 || position > this.length -1) {
        return false
    }
    else {
        //2
        if (position === 0) {
            this.head = this.head.next;
        }
        //3
        else {
            let currentNode = this.head;
            let lastNode = null;
            let index = 0;
            while(index++ < position) {
                lastNode = currentNode;
                currentNode = currentNode.next;
            }
            lastNode.next = currentNode.next;
        }
        this.length --;
        return currentNode;
    }
}
